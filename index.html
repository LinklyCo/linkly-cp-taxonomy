Critical strategic questions - resolve before pricing phase
Q1
Does Connect (now the umbrella for On-Prem, API and SoftPOS modes) survive as a separate product, or is it Core with the gateway toggled off?
Core is defined by the IP Gateway being standard. Connect = connectivity only, no gateway, regardless of mode. With Bridge merged in, Connect now spans traditional middleware, third-party terminal API, and SoftPOS - one product story, three delivery modes. If that distinction holds commercially, Connect is a real product. If not, the modes become pricing variants of Core with a gateway line-item waived. This question now also absorbs the old Q7 (Bridge cannibalisation) - same boundary, one decision.
Q2
What is the Orbit base price relative to Core and PayApp legacy rates?
Must be at or above equivalent legacy rates to avoid cannibalising Core revenue on migration. Current Orbit rates: ANZ $15.36 flat, aggregators $14 to $18.50 tiered. PayApp rates: $10.60 to $17.45. The migration pricing model needs to be defined. Q4
Q3
Is Linkly Cloud an add-on across all products, or does it become standard at some point?
Linkly Cloud is OS-agnostic, lower support burden, direction of travel. On-Prem is where current volume lives. A surcharge incentivises migration but creates friction with large estates (CBA, NAB, ANZ). The answer also determines whether On-Prem has a sunset date.
Q4
What is the migration commercial model for PayApp to Orbit?
Transition pricing, hostlink re-certification fee, step-up schedule? This affects approx $1.7M of current PayApp revenue and the speed of the migration programme. Does Linkly charge for re-certification, or absorb it to accelerate the move?
Q5
How should Standalone be priced relative to Integrated?
Current data shows Standalone at $2.56 to $7.50 vs $15.36 for Integrated (ANZ). The gap reflects the absence of the middleware component. Does this hold as the standard differential, or should it be reset when the new taxonomy is formalised?
Q6
What is the revenue model for Connect's API mode (was: Bridge)?
Per-transaction, per-connection, or platform fee? Who does Linkly contract with - the POS vendor, the merchant, or the third party terminal provider? The On-Prem mode is per-terminal/month; API mode probably should not be. Until this is answered, the API mode cannot enter the pricing phase, which means consolidated Connect cannot be priced as one product yet.
Q7 · Resolved via Q1
Does Bridge cannibalise Core or open a new market?
Resolved by the Connect/Bridge merge. Bridge is now the API mode within Connect - so the cannibalisation question collapses into Q1 (Connect-vs-Core boundary). Whatever answers Q1 for the On-Prem mode answers it for the API and SoftPOS modes too: same product, same commercial fence.
Q8
What is the Clover status in the Connect API mode?
A Clover licence line appears in the Pricing sheet - more real than previously assumed. Confirm whether this is a live commercial relationship, a draft term sheet, or a dropped integration before the API mode is taken any further commercially. Clover is the largest known prospective API-mode customer after Square.
Q9
Is SoftPOS its own Connect mode, and does it share API mode's commercial model?
SoftPOS (COTS Android device acting as a payment terminal) currently sits as the third Connect mode. Confirm: is it commercially distinct from API mode, or is it a sub-flavour of it? If distinct, it needs its own pricing answer (separate from Q6). If a sub-flavour, Q6 covers it. This affects whether Connect has 2 or 3 commercial models bundled inside it.
